## 4. 物理自洽检查

### 4.1 自洽部分

#### 链路 1：触发 -> 执行 -> 实际闪现

```text
flash_eval_trigger_count_sum = 10385
flash_execute_count_sum = 1766
flash_count_sum = 1224
```

满足：

```text
10385 >= 1766 >= 1224
```

该链路是自洽的。

#### 链路 2：有效 + 无效 = 总闪现

```text
effective_flash_count_sum = 1019
ineffective_flash_count_sum = 205
flash_count_sum = 1224
```

满足：

```text
1019 + 205 = 1224
```

该链路也是自洽的。

### 4.2 不自洽部分

本轮还统计到：

```text
danger_flash_count_sum = 297
safe_flash_count_sum = 857
```

则：

```text
danger_flash_count_sum + safe_flash_count_sum = 1154
```

但：

```text
flash_count_sum = 1224
```

两者不相等：

```text
1154 != 1224
差值 = 70
```

这说明本轮原始日志中，至少有 `70` 次闪现：

- 已经记入 `flash_count`
- 但没有被归类到 `danger_flash_count`
- 也没有被归类到 `safe_flash_count`

因此，这里的问题不是简单的汇总脚本加错，而是原始日志字段本身就存在分类不完整。
